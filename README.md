Campus RAG Assistant (local RAG with citations)

What it does
- Upload PDFs/DOCX/TXT/MD into data/raw_docs
- Index them into a Qdrant vector database using bge-m3 embeddings
- Ask questions in a Streamlit UI
- The answer is generated by a local LLM via Ollama, using only retrieved context
- The UI shows sources (filename, page, similarity score, chunk id)
- If evidence is missing, it refuses: "I don't know based on the provided documents."

Architecture (basic RAG)

[Documents] -> [Text extraction] -> [Chunking] -> [Embeddings: bge-m3] -> [Qdrant]
                                                       |
                                                       v
User Question -> [Query embedding] -> [Top-K retrieval] -> [Prompt w/ context] -> [Ollama LLM] -> Answer + citations

Quickstart (when you’re ready)
1) Start Qdrant
   docker compose up -d

2) Create venv and install
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt

3) Start Ollama and pull a model
   ollama serve
   ollama pull phi3
   (or) ollama pull llama3

4) Configure env vars (optional)
   Copy .env.example to .env and edit:
   - OLLAMA_MODEL=phi3 or llama3
   - TOP_K, CHUNK_CHARS, OVERLAP_CHARS, etc.

5) Run the UI
   streamlit run app/ui_streamlit.py

How indexing works
- Each file is hashed with sha256.
- A manifest is saved at data/processed/index_manifest.json.
- If the same filename has the same hash, the index step skips it (no re-embedding).

How citations work
- The model is instructed to cite sources in square brackets.
- For PDFs, it should cite like: [filename p.X]
- For non-PDF, it should cite like: [filename]
- The UI also shows exactly which chunks were retrieved.

Troubleshooting
- Qdrant not running:
  - open http://localhost:6333 in your browser
  - if it fails, run: docker compose up -d

- Ollama not running:
  - run: ollama serve
  - verify: http://localhost:11434
  - make sure your model exists: ollama list

- Indexing shows 0 chunks:
  - the document may be scanned (image-only PDF). This project does not do OCR.

Evaluation
- Run:
  python eval/eval_runner.py
- Outputs go to eval/results.jsonl
- It prints avg retrieval time, avg generation time, and how often it refused.
